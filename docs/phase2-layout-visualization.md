- **Width**: Full width (no columns)
- **Content**: Only "Deine Konfiguration" column
- **Removed**: Documents and Sections columns
- **Function**: Plan selection, program connection, product type selection

- **Position**: Above preview, below TemplateOverviewPanel
- **Layout**: Horizontal scrollable row
- **Style**: 
  - Compact card design
  - Horizontal scroll with overflow
  - Similar to current document cards but optimized for horizontal layout
- **Features**:
  - Add document button (first card)
  - Document cards with checkboxes
  - Edit, toggle, open content actions
  - Origin badges

## Key Design Decisions

### 1. Documents Above Preview
- **Rationale**: Documents are selected once and affect which sections are shown
- **Benefit**: Clear visual hierarchy - documents filter sections
- **UX**: Horizontal scroll keeps it compact while showing all documents

### 2. Sections Merged with Sidebar
- **Rationale**: Sections are navigated frequently, template management is contextual
- **Benefit**: Single source of truth for sections - navigation and management in one place
- **UX**: Reduces cognitive load, all section operations in sidebar

### 3. Configuration Standalone
- **Rationale**: Configuration is high-level, affects entire plan
- **Benefit**: Clear separation of concerns
- **UX**: Configuration is less frequently changed, doesn't need to be in workspace

## Implementation Considerations

### Documents Bar Component
- New component: `DocumentsBar.tsx`
- Props: documents, onToggle, onEdit, onSelect, etc.
- Horizontal scroll container
- Compact card design (smaller than current cards)

### Enhanced Sidebar Component
- Extend existing `Sidebar.tsx`
- Add template management UI to each section item
- Integrate edit forms (modal or inline)
- Maintain current navigation functionality

### TemplateOverviewPanel Changes
- Remove `DesktopTemplateColumns` usage
- Keep only `DesktopConfigurator`
- Reduce height/space usage
